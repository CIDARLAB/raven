RavenCAD Overview

RavenCAD is a DNA assembly design and planning tool. The tool accepts as its inputs a list of target parts and a parts library. For any set of target parts, RavenCAD determines an optimized plan for assembling all selected target parts with a particular DNA assembly method while optimizing for time, efficiency and cost.

A unique feature of RavenCAD is its ability to incorporate experimental data to drive the assembly solution. RavenCAD allows the user to input efficiency information for a variety of parameters and bias or force certain assembly intermediates or overhangs from appearing in the assembly solution. The tool comes with a set of default parameters that have been experimentally verified, but these values can be modified to the user’s specifications. This user-interactivity based on experimental data allows users to tailor the assembly solutions to lab- and experiment-specific conditions and recalculate increasingly effective solutions.

Assembly Methods

RavenCAD broadly classifies cloning-based assembly methods into three categories: binary assembly, modular one-pot assembly and scarless one-pot assembly. BioBricks(TM) assembly [4] and its derivatives are supported binary assembly methods. CPEC [6], SLIC [5], Gibson [2] and GoldenGate [7] assembly are all supported scarless one-pot assembly methods. MoClo [3] is a supported modular one-pot assembly method. Each of the listed supported assembly methods has additional technology-specific constraints.

Additional cloning-based assembly method and chemical synthesis assembly are not currently supported. However, RavenCAD’s core algorithm is highly generalizable and new methods can be added with relative ease. Please contact the developers if you are interested in an assembly method that is not currently supported.

Algorithms

Raven breaks the DNA assembly problem into two major parts: selecting and generating optimized overhangs and performing all cloning reactions necessary to achieve the final construct. The tool first determines an optimized set of assembly steps, and then it optimizes overhang selection; it is assumed that performing a successful assembly step is more costly and time consuming than performing a PCR reaction. This strategy is applied to binary, one-pot scarless and one-pot modular assembly methods to produce fast, modular and efficient assembly plans.

Sometimes an experimentalist observes higher or lower assembly efficiencies than realized in the literature due to a number of experimental conditions such as cell competency, specific reaction conditions and other factors. With RavenCAD, a user is able to adjust the default efficiency numbers guiding the efficiency optimizations. A user may also wish to bias certain intermediates from appearing or not appearing, with or without mandatory inclusion or exclusion. For example, a user might wish to bias an assembly towards intermediates that fit a certain substructure such as a transcriptional unit. Another user might want to bias against performing PCR on large parts or parts with hairpins or other problematic structures. A user might also encounter unsuccessful cloning steps or PCRs or wish to test specific intermediate parts or part junctions. In all of these cases, it is imperative that these intermediates and/or part junctions are either biased or forced to appear or not appear. Raven optimizes around these additional constraints unless they are so restrictive that no valid solution can be found. RavenCAD allows for lists of intermediates or overhangs to be input as optional parameters for the algorithm. Intermediate parts and overhangs can be biased for (recommended), biased against (discouraged), forced or forbidden. These parts and overhangs can be specified either manually in the GUI or automatically via Eugene [8] scripts.

Integrated Tool Chain

RavenCAD is a ‘stand-alone’ tool, meaning it does not depend on any other tools or databases to provide optimized assembly plans. However, this tool is also connected to a larger tool-chain in the Clotho design automation platform. The tool chain is a top-down design flow and begins with the specification of device behavior. One tool in the Clotho design flow that can do this is Eugene [8]. Eugene can be used to generate a set of target devices based on libraries of parts and device design rule sets. Once all target devices are determined, RavenCAD determines the best way to build all target parts. These output instructions can be given to a liquid handling robot to automatically assemble these devices from its resources. This flow allows a user to simply specify the behavior of a set of devices they would like to build and what resources they have and have the devices automatically constructed with liquid handling robots.

Help

What do I need to use RavenCAD?
A user must minimally provide a list of target parts and all DNA parts that are needed to build the target parts. Part sequences are not mandatory, but no sequence-level optimization can be performed on parts without sequence.
What do I input into RavenCAD?
A user may input a list of target parts and library of existing parts or input a collection of annotated genbank files.
What does RavenCAD output?
RavenCAD outputs several types of information:
  1. A graphical representation of an optimized assembly plan for the selected target parts.
  2. A set of assembly instructions for a liquid handling robot (computer-readable)
  3. A set of assembly instructions for a human user (human-readable)
    1. List of all primers to needed to accomplish assembly
    2. List of all PCR reactions to perform (all part-primer sets)
    3. List of cloning steps to perform with a graphical picture of each individual step
What information is mandatory to run RavenCAD and what information is optional?
  • Mandatory: Target parts, parts composing all target parts
  • Optional: Existing overhangs for each input part; existing assembly vectors; complete parts library assembly efficiency information for a specific cloning method; recommended, discouraged, required, forbidden intermediate parts and/or overhangs
